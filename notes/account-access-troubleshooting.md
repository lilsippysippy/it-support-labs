# Account Access Troubleshooting

This file documents common account access troubleshooting steps for IT support practice.

## Purpose

The goal of this lab is to understand how IT support technicians troubleshoot user login, password, MFA, permission, and account access issues.

## Common Account Access Issues

Users may report issues such as:

* Cannot sign in
* Password not working
* Account locked
* MFA prompt not working
* User cannot access email
* User cannot access a shared folder
* User cannot access Microsoft 365 apps
* User does not have permission to open a file or application
* User recently changed devices and cannot sign in

## Basic Troubleshooting Process

1. Confirm the user’s identity according to company policy.
2. Confirm the exact system, app, or resource they are trying to access.
3. Ask for the exact error message.
4. Check whether the issue affects one user or multiple users.
5. Confirm whether the user is entering the correct username.
6. Check for password reset or account lockout needs.
7. Check MFA status or authentication method.
8. Check group membership or permissions.
9. Document all troubleshooting steps.
10. Escalate if the issue requires administrator approval or higher access.

## Password Reset Concepts

Password reset support may involve:

* Verifying the user
* Resetting a temporary password
* Having the user create a new password
* Confirming the user can sign in
* Documenting the completed reset

## MFA Concepts

Multi-factor authentication, or MFA, adds an extra layer of security during sign-in.

Common MFA issues include:

* User changed phones
* Authentication app is not working
* User is not receiving a code
* User is locked out
* MFA method needs to be reset
* User is using the wrong account

## Permission Troubleshooting

If a user cannot access a file, folder, app, or shared resource, possible causes include:

* User is not in the correct group
* User does not have the correct license
* File or folder permissions are missing
* Account is disabled or locked
* Access request has not been approved
* The resource has been moved or deleted

## Example Scenario

### Problem

A user says they cannot access a shared folder.

### Steps to Check

1. Confirm the folder path or SharePoint location.
2. Ask what error message appears.
3. Confirm whether other users can access the same folder.
4. Check whether the user recently changed roles or teams.
5. Check permissions or group membership.
6. Confirm whether access should be granted.
7. Escalate if approval is needed.
8. Document the issue and final result.

## Documentation Notes

When documenting an account access issue, I should include:

* User affected
* System or resource affected
* Error message
* Whether the issue affects one user or multiple users
* Identity verification completed according to policy
* Troubleshooting steps taken
* Password, MFA, or permission notes
* Resolution or escalation details

## What I Am Learning

I am learning how account access issues connect to identity, security, permissions, MFA, and user support. This is important for help desk, desktop support, IT support, and production support roles.

## Safety Note

I should never document or upload passwords, MFA codes, private account details, screenshots with personal information, or internal company access information.
